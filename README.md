# FER2013 — სახის გამომეტყველების ამოცნობა

## შესავალი

ეს ანგარიში ასახავს ექსპერიმენტების სერიას, რომლებიც ჩატარდა Kaggle-ის სახის გამომეტყველების ამომცნობ (FER2013) მონაცემთა ბაზაზე. მიზანი იყო 48 x 48 განზომილებიანი, ნაცრისფერი  სახის სურათების კლასიფიკაცია 7 ემოციურ კატეგორიად: ბრაზი, ზიზღი, შიში , სიხარული, სევდა, გაკვირვება და ნეიტრალური ემოცია . მონაცემთა ბაზა შეიცავს 35 887 სურათს, რომელიც დაყოფილია 28 709 სასწავლო (training), 3 589 ვალიდაციურ (validation) და 3 589 სატესტო (test) ნიმუშად.


ამ დავალების მთავარი გამოწვევა იყო არა მხოლოდ მაღალი სიზუსტის მიღწევა, არამედ მოდელის განვითარების სრული სასიცოცხლო ციკლის მიზანმიმართული დემონსტრირება, underfitting -დან დაწყებული, overfitting-ის იდენტიფიცირებითა და რეგულარიზაციის ტექნიკების გამოყენებით საუკეთესო განზოგადების უნარის მქონე მოდელის მიღებამდე. ყველა ექსპერიმენტი რეალურ დროში კონტროლდებოდა Weights & Biases პლატფორმის გამოყენებით.

## ექსპერიმენტი 1: Underfitting TinyMLP-ის გამოყენებით

**Run:** `run1_tiny_mlp`

პირველ ექსპერიმენტში გამოვიყენეთ მიზანმიმართულად მინიმალისტური MLP (Multi-Layer Perceptron), რომელსაც მხოლოდ 147975 პარამეტრი ჰქონდა:

```
Flatten → Linear(2304→64) → ReLU → Linear(64→7)
```

**შედეგები 30 ეპოქის შემდეგ:**
- Train accuracy: **38%**
- Val accuracy: **40%**
- Train/Val gap: **0%**

**რას გვიჩვენებს გრაფიკები:** სატრენინგო და ვალიდაციური დანაკარგები (loss) ნელა მცირდება და ადრეულ ეტაპზევე სტაბილურდება (plateau). სატრენინგო და ვალიდაციურ სიზუსტეებს შორის სხვაობა თითქმის არ არის, მეტიც, ვალიდაციური სიზუსტე ოდნავ მაღალიცაა სატრენინგოზე. ეს არის underfitting-ის კლასიკური მახასიათებელი.

**რატომ მოხდა:** ამის ორი მიზეზი არსებობს. პირველი, 48 x 48 ზომის სურათის ვექტორად გაბრტყელება (flattening) მთლიანად ანადგურებს პიქსელებს შორის არსებულ სივრცით კავშირებს. სახეს აქვს თავისი სტრუქტურა: თვალები ცხვირის ზემოთაა, პირი კი ქვემოთ, თუმცა MLP თითოეულ პიქსელს განიხილავს როგორც დამოუკიდებელ მახასიათებელს მეზობელი პიქსელების გათვალისწინების გარეშე. მეორე, მხოლოდ 64 ფარული ნეირონით მოდელს უბრალოდ არ აქვს საკმარისი მოცულობა და მეხსიერება, რომ ისწავლოს გადაწყვეტილების მიღების საზღვრები 7 რთულ ემოციურ კლასს შორის. მოდელი გარკვეულწილად სწავლობს (40% ბევრად აღემატება შემთხვევითი გამოცნობის ალბათობას, 14%-ს), მაგრამ მალევე აღწევს მაქსიმუმს და სწავლების დროის გაზრდის მიუხედავად, პროგრესს ვეღარ აკეთებს.მოსაყოფად.

**დასკვნა:**  ეს მოდელი განიცდის underfitting-ს. პრობლემა არქიტექტურულია და არა სწავლების ხანგრძლივობის ან სწავლის სიჩქარის (learning rate) ცვლილების საკითხი.

## ექსპერიმენტი 2: საბაზისო CNN მოდელი SmallCNN-ით

**Run:** `run2_small_cnn`

მეორე ექსპერიმენტში MLP ჩავანაცვლე მცირე ზომის კონვოლუციური ნეირონული ქსელით (CNN):

```
Conv2d(1→32) + ReLU + MaxPool → Conv2d(32→64) + ReLU + MaxPool → Linear(9216→256) → Linear(256→7)
```

**შედეგები 30 ეპოქის შემდეგ:**
- Train accuracy: **67%**
- Val accuracy: **55%**
- Train/Val gap: **12%**

**რას გვიჩვენებს გრაფიკები:** TinyMLP-სთან შედარებით მკვეთრი გაუმჯობესებაა, ვალიდაციის სიზუსტე მყისიერად იზრდება 40%-დან 55%-მდე. თუმცა, დროთა განმავლობაში მრუდებს შორის სხვაობა იკვეთება: სატრენინგო დანაკარგი (train loss) სტაბილურად კლებულობს, მაშინ როცა ვალიდაციური დანაკარგი (val loss) მე-15 ეპოქის გარშემო ჩერდება. მე-30 ეპოქისთვის სხვაობა სატრენინგო და  ვალიდაციურ სიზუსტეებს შორის 12%-ს აღწევს.

**რატომ მოხდა:** კონვოლუციური შრეები ინარჩუნებენ სივრცით სტრუქტურას. 3 x 3 ფილტრი გადაადგილდება სურათზე, აფიქსირებს ლოკალურ პატერნებს, კიდეებს, მოსახვევებს, ტექსტურებს, და ქმნის სახის მახასიათებლების იერარქიულ რეპრეზენტაციას. ეს სწორი ინდუქციური bias -ია ვიზუალური მონაცემებისთვის. თუმცა, ყოველგვარი რეგულარიზაციის გარეშე, მოდელი სწავლის გაგრძელებასთან ერთად იწყებს სასწავლო მაგალითების ზეპირად დამახსოვრებას, რაც overfitting-ის ადრეული ნიშანია.

**დასკვნა:** CNN ფუნდამენტურად უკეთესია გამოსახულებებთან დაკავშირებული ამოცანებისთვის, ვიდრე MLP. თუმცა, მცირე ზომის CNN-იც კი რეგულარიზაციის გარეშე იწყებს ზედმეტ დაზეპირებას. 12%-იანი სხვაობა საგანგაშო ნიშანია.ია.

## ექსპერიმენტი 3: მიზანმიმართული Overfitting LargeCNN-ის გამოყენებით

**Run:** `run3_large_cnn_overfit`

მესამე ექსპერიმენტში მკვეთრად გავზარდე CNN-ის მასშტაბები, თუმცა ყოველგვარი **რეგულარიზაციის გარეშე:**

```
Conv(1→64) + Conv(64→128) + Conv(128→256) → Linear(9216→1024) → Linear(1024→512) → Linear(512→7)
```

> არ გამომიყენებია არც Dropout, არც BatchNorm და არც წონის კლება.

**შედეგები 50 ეპოქის შემდეგ:**
- Train accuracy: **93%**
- Val accuracy: **59%**
- Train/Val gap: **34%**

**რას გვიჩვენებს გრაფიკები:** კლასიკური overfitting. ამაზე მიგვანიშნებს ორი უმთავრესი სიგნალი:

1. სასწავლო სიზუსტე მე-5 ეპოქაზე არსებული 53%-დან მე-50 ეპოქისთვის 93%-მდე იზრდება. სავალიდაციო სიზუსტე კი მე-10 ეპოქისთვის აღწევს 59%-ს და შემდეგ თითქმის აღარ იცვლება, ის სრულ plateu-ს განიცდის, მაშინ როცა სატრენინგო სიზუსტე აგრძელებს ზრდას.
2. ვალიდაციური დანაკარგის აცილება: მე-10 ეპოქის შემდეგ (სადაც ვალიდაციური დანაკარგი მინიმუმზეა 1.17), ვალიდაციური დანაკარგი უწყვეტად იწყებს ზრდას და მე-50 ეპოქისთვის 2.24-ს აღწევს. ამავდროულად, სატრენინგო დანაკარგი 0.20-მდე ეცემა. ეს ორი მრუდი საპირისპირო მიმართულებით მოძრაობს, რაც ზედმეტი მორგების ყველაზე ნათელი ინდიკატორია.

**რატომ მოხდა:** 10 მილიონი პარამეტრით და დამახსოვრების პრევენციის მექანიზმების გარეშე, მოდელს აქვს საკმარისი მოცულობა, რომ ზეპირად დაიმახსოვროს თითოეული სასწავლო სურათი. ნაცვლად იმისა, რომ ისწავლოს ზოგადი პატერნები, მაგალითად: „აწეული ტუჩის კუთხეები = სიხარულს“, მოდელი იმახსოვრებს კონკრეტულ პიქსელურ განლაგებებს სასწავლო მონაცემებიდან. ხოლო, მაშინ უკვე როდესაც ის ხედავს ახალ სურათებს ვალიდაციისას, ეს დამახსოვრებული შაბლონები აღარ მუშაობს.


**Backward checking:**WandB-ის გრაფიკზე დაკვირვებით, ადრეული გაჩერების ოპტიმალური წერტილი იყო მე-10 ან მე-12 ეპოქა, სადაც ვალიდაციური დანაკარგი ყველაზე დაბალი იყო. ამ წერტილის მიღმა ტრენინგი აქტიურად აზიანებს მოდელის განზოგადების უნარს.


**დასკვნა:** მეტი პარამეტრი რეგულარიზაციის გარეშე იწვევს ზეპირად დამახსოვრებას და ნაკლებ სწავლას.

## ექსპერიმენტი 4: საუკეთესო მოდელი RegularizedCNN-ით

**Run:** `run4_regularized_cnn`

მეოთხე ექსპერიმენტში გამოვიყენე ზუსტად იგივე არქიტექტურა, რაც მე-3 ექსპერიმენტში, მაგრამ დავამატე რეგულარიზაციის სამი ტექნიკა:

```
Conv(1→64)+BatchNorm + Conv(64→128)+BatchNorm + Conv(128→256)+BatchNorm
→ Linear(9216→1024) + Dropout(0.5)
→ Linear(1024→512) + Dropout(0.5)
→ Linear(512→7)
```

გარდა ამისა: L2 weight decay (1e-4) ოპტიმიზატორში.

**შედეგები 30 ეპოქის შემდეგ:**
- Train accuracy: **69%**
- Val accuracy: **63%**
- Train/Val gap: **6%**

**რას გვიჩვენებს გრაფიკები:** მე-3 ექსპერიმენტთან შედარებით ტრანსფორმაცია შთამბეჭდავია. სიზუსტის სხვაობა 34%-დან სულ რაღაც 6%-მდე შემცირდა. რაც მთავარია, val loss აგრძელებს კლებას მე-30 ეპოქამდე — ის არასოდეს განიცდის დივერგენციას. მოდელი რეალურად აუმჯობესებს განზოგადების უნარს და არა უბრალოდ იმახსოვრებს მონაცემებს.

**რატომ მუშაობს რეგულარიზაცია:**
- **BatchNorm** — ახდენს შრეების output ნორმალიზებას, რაც უზრუნველყოფს სწავლების სტაბილურობას და საშუალებას აძლევს მოდელს ისწავლოს უფრო სწრაფად, გრადიენტების exploding-ის ან გაქრობის (vanishing) გარეშე. ის ასევე მცირე რეგულარიზაციას ახდენს ბეტჩის სტატისტიკაში ხმაურის (noise) შეტანით.
- **Dropout(0.5)** — შემთხვევითობის პრინციპით თიშავს ნეირონების 50%-ს სწავლების თითოეულ ბიჯზე. ეს ხელს უშლის რომელიმე კონკრეტული ნეირონის ზედმეტად სპეციალიზაციას. ქსელი იძულებულია ისწავლოს დუბლირებული, განაწილებული რეპრეზენტაციები, რაც ბევრად უკეთ გენერალიზდება.
- **L2 weight decay** — აჯარიმებს დიდ წონებს და უბიძგებს ოპტიმიზატორს უფრო მარტივი გადაწყვეტებისკენ, სადაც წონები მცირე და თანაბრად გადანაწილებულია.

**დასკვნა:** იგივე 10-მილიონიანი მოდელი, რომელმაც კატასტროფული overfitting აჩვენა მე-3 ექსპერიმენტში, კარგად გენერალიზდება რეგულარიზაციის პირობებში. პარამეტრების სიმრავლე თავისთავად არ იწვევს overfitting-ს, ამას რეგულარიზაციის ნაკლებობა განაპირობებს.

## ჰიპერპარამეტრების ექსპერიმენტები

### Learning Rate Sweep

## დასკვნები

ეს პროექტი თვალსაჩინოდ აჩვენებს მოდელის განვითარების სრულ პროცესს გამოსახულებების კლასიფიკაციისთვის:

1. **არქიტექტურა მნიშვნელოვანია ზომაზე მეტად.** TinyMLP 147 ათასი პარამეტრით ჩავარდა არა იმიტომ, რომ პატარა იყო, არამედ იმიტომ, რომ არასწორი არქიტექტურა ჰქონდა ვიზუალური მონაცემებისთვის. SmallCNN-მა კი, 2.4 მილიონი პარამეტრითა და სწორი სტრუქტურით (კონვოლუციებით), მყისიერად აჯობა მას.
2. **Overfitting-ის  შექმნაც მარტივია და მისი დიაგნოსტიკაც.** რეგულარიზაციის გარეშე შრეებისა და ნეირონების დამატება გარანტირებულად იწვევს overfitting-ს , დიდ და მზარდ სხვაობას სასწავლო და სავალიდაციო სიზუსტეებს შორის და val loss-ის ზრდას მაშინ, როცა train loss მცირდება.
3. **რეგულარიზაცია მუშაობს.** BatchNorm + Dropout + L2 კომბინაციამ შეამცირა სხვაობა სასწავლო და სავალიდაციო სიზუსტეებს შორის 34%-დან 6%-მდე იმავე არქიტექტურაზე და გაზარდა ვალიდაციური სიზუსტე 59%-დან 63%-მდე.
4. **ჰიპერპარამეტრებს დიდი მნიშვნელობა აქვს.** სწორად შერჩეული სწავლის კოეფიციენტი (0.001) და ოპტიმიზატორი (Adam) სწავლებას ხდის სტაბილურსა და ეფექტურს. არასწორმა კოეფიციენტმა (0.01) თითქმის 14 ეპოქა ტყუილად დააკარგვინა მოდელს, სანამ ის სწავლას დაიწყებდა.
5. **მიზანია განზოგადება, არა train accuracy.** მოდელი, რომლის სასწავლო სიზუსტე 93%-ია, ხოლო ვალიდაციური 59%, ბევრად უარესია მოდელზე, რომელსაც აქვს 69%-იანი სასწავლო და 63%-იანი სავალიდაციო სიზუსტე. გადამწყვეტია მოდელის მუშაობის ხარისხი იმ მონაცემებზე, რომლებიც მას მანამდე არ უნახავს.
